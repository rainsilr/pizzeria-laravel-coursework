Учебное веб-приложение для курсовой работы по ПМ.09 «Проектирование, разработка и оптимизация веб-приложений».

## Описание проекта

Проект представляет собой сайт пиццерии с публичным просмотром меню и защищенным управлением позициями меню после авторизации.

Основная сущность проекта: пицца.
Связанная сущность: категория пиццы.

В приложении реализованы список пицц, детальная страница, создание, редактирование и удаление записей. Неавторизованные пользователи могут просматривать публичные страницы, а управление меню доступно только после входа.

## Технологии

- PHP 8.3
- Laravel 13
- MySQL
- Blade
- Laravel Breeze
- Eloquent ORM
- Vite
- Tailwind CSS
- Nginx

## Функциональные возможности

- регистрация и вход пользователей через Laravel Breeze;
- публичная главная страница;
- публичный список пицц с выводом связанных категорий;
- детальный просмотр одной пиццы;
- создание, редактирование и удаление пицц для авторизованных пользователей;
- серверная валидация форм;
- поиск по названию и описанию;
- фильтрация по категории;
- пагинация списка;
- flash-сообщения после операций;
- заполнение базы тестовыми данными через сидер.

## Структура базы данных

В проекте используются связанные таблицы:

- `categories` — категории пицц;
- `pizzas` — позиции меню.

Связь между таблицами:

- одна категория содержит много пицц;
- одна пицца принадлежит одной категории.

В моделях настроены отношения Eloquent:

- `Category::pizzas()`;
- `Pizza::category()`.

## Локальное развертывание

1. Установить зависимости PHP:

```bash
composer install
```

2. Установить зависимости Node.js:

```bash
npm install
```

3. Создать файл окружения:

```bash
cp .env.example .env
```

4. Сгенерировать ключ приложения:

```bash
php artisan key:generate
```

5. Создать базу данных MySQL `pizzeria_laravel` и настроить подключение в `.env`:

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=pizzeria_laravel
DB_USERNAME=root
DB_PASSWORD=
```

6. Выполнить миграции и заполнить базу тестовыми данными:

```bash
php artisan migrate --seed
```

7. Собрать фронтенд:

```bash
npm run build
```

8. Запустить приложение:

```bash
php artisan serve
```

После запуска приложение будет доступно по адресу `http://127.0.0.1:8000`.

## Развертывание на сервере

При размещении проекта на сервере необходимо:

1. Склонировать репозиторий.
2. Установить зависимости Composer без dev-пакетов.
3. Настроить `.env`.
4. Создать MySQL-базу данных.
5. Выполнить миграции и сидеры.
6. Собрать frontend.
7. Настроить Nginx на директорию `public`.
8. Выполнить оптимизацию Laravel.

Основные команды:

```bash
composer install --no-dev --optimize-autoloader
npm install
npm run build
php artisan key:generate
php artisan migrate --seed --force
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

Для веб-сервера корневой директорией сайта должна быть папка:

```text
public
```
